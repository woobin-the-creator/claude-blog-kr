# AI 인사이트를 얻는 소스 모음

> 작성일: 2026-06-25
> 출처: 영어권 커뮤니티(Hacker News, Reddit) + 큐레이션 글 종합. 맨 아래 Sources 참고.
> 전제: 이미 **Claude 공식 블로그/유튜브**, **Cursor 공식 유튜브**를 구독 중 → "1차 소스(만든 사람이 직접 말하는 것)"는 확보된 상태. 아래는 그 사각지대를 메우는 보강용.

## 한눈에 — 3종류만 추가하면 충분

1. **종합 뉴스레터 1개** — 매일의 전체 흐름
2. **연구 해설 유튜브/X 1~2개** — 왜 중요한지 깊이
3. **커뮤니티 1개** — 현장 실무자들의 날것 반응

---

## 1. 뉴스레터 (이메일 자동 배달, 손이 가장 안 감)

| 이름 | 성격 | 추천 대상 |
|---|---|---|
| **TLDR AI** ⭐ | 매일, 짧고 기술 중심 | 개발자 1순위 |
| Ahead of AI (Sebastian Raschka) | 주간, LLM 구조/구현 심화 | 모델 내부 원리 |
| Import AI (Jack Clark) | 주간, 정책·거시 동향 | 산업 흐름 |
| The Batch (Andrew Ng) | 주간, 교육적 | 입문~중급 |
| The Rundown / Superhuman | 매일, 대중적·툴 위주 | 가볍게 넓게 |

**추천: TLDR AI 하나.** 매일 오지만 5분 안에 읽히고, 마케팅보다 기술·릴리스 중심.
정설은 "매일 1개 + 주간 심화 1개" — 심화는 Ahead of AI가 잘 맞음.

## 2. 유튜브 (공식 채널 외 보강)

| 채널 | 성격 |
|---|---|
| **AI Explained** ⭐ | 벤치마크 직접 검증, 과대광고 거름. 빈도 낮지만 신뢰도 최상 |
| Yannic Kilcher | 논문 직접 해부, 가장 기술적 |
| Two Minute Papers | 최신 논문 2~4분 요약 |
| Lex Fridman | 연구자/창업자 장시간 인터뷰 |
| Matt Wolfe (Future Tools) | 신규 AI 툴 주 2~3회 |

**추천: AI Explained.** 공식 채널로 "발표"는 챙기고 있으니, 그게 실제 사실인지 독립 검증해주는 채널이 가장 큰 보완.

## 3. X(트위터) — 1차 정보가 공식 블로그보다 며칠 빠름

| 계정 | 왜 |
|---|---|
| **@karpathy** (Andrej Karpathy) ⭐ | LLM 직관·방향성 |
| **@simonw** (Simon Willison) ⭐ | 프롬프트 인젝션·로컬 LLM·실전 보안 |
| @rasbt (Sebastian Raschka) | 구현 중심 |
| @emollick (Ethan Mollick) | 데이터 기반 실험, "AI가 뭘 잘하고 못하나" |
| @fchollet (François Chollet) | 똑똑한 회의론자, 균형추 |

**추천: @simonw + @karpathy.** 한 명은 실전/보안, 한 명은 큰 그림 — 도구 실사용자에게 가장 균형 잡힌 조합.

## 4. 커뮤니티 (실무자들의 날것 반응)

| 곳 | 성격 |
|---|---|
| **Hacker News** ⭐ | 기술 구매자/엔지니어의 솔직한 토론, 댓글이 핵심 |
| **r/LocalLLaMA** ⭐ | 오픈 모델·양자화·로컬 실행·GPU 셋업, 기술 토론 최고로 꼽힘 |
| r/MachineLearning | 논문·연구 비평 |

**추천: Hacker News + r/LocalLLaMA.** HN은 "이 도구가 실제로 쓸 만한가", LocalLLaMA는 "내 PC에서 직접 돌리는" 관점.
HN 평가: "공식 발표보다 진짜 통증(pain)이 먼저 보인다."
