Claude Code가 이제 아티팩트(artifacts)를 지원합니다

2026년 6월 18일 · 읽는 데 약 5분 · 카테고리: Product announcements 원문: Claude Code now supports artifacts (한글 번역본)

오늘부터 Claude Code는 작업 진행 상황을 아티팩트(artifact)로 담아낼 수 있습니다. 아티팩트는 Claude Code의 작업을 살아 움직이고 공유 가능한 시각적 페이지로 바꿔 줍니다. PR 설명(walkthrough), 시스템 해설(explainer), 대시보드, 릴리스 체크리스트 등이 여기에 해당하며, 세션이 일을 진행하는 동안 스스로 갱신됩니다.

Claude Code 세션은 장애(incident) 조사부터 서비스 리팩터링, 수개월치 데이터 분석까지 그 범위가 다양합니다. 아티팩트는 이 작업을 누구나 열어 살펴볼 수 있는 웹 페이지로 옮겨 줍니다. 예를 들어 풀 리퀘스트(pull request) 설명, 필터링하고 정렬할 수 있는 대시보드, 또는 작업이 끝나는 대로 스스로 채워지는 릴리스 체크리스트가 될 수 있습니다. 아티팩트는 공동 작업물에 대해 협업하기 쉽게 해 주며, 그래서 팀은 진행 상황을 보고하는 데 들이는 시간을 줄이고 만드는 데 더 많은 시간을 쓸 수 있습니다.

세션의 컨텍스트를 기반으로 만듭니다

Claude Code는 세션의 전체 컨텍스트, 즉 코드베이스, 커넥터(connector), 그리고 대화 자체를 모두 활용해 아티팩트를 만듭니다. 하나의 장애 페이지는 코드에서 실패한 테스트와 그 뒤에 있는 함수, 연결된 모니터링 도구에서 가져온 오류 급증(error spike), 그리고 방금 실행한 세션의 근본 원인(root-cause) 추론을 한데 모을 수 있습니다. 아티팩트를 쓰면 데이터 소스를 연결하거나 인프라를 구축할 필요가 없습니다. 페이지 하나를 요청하면, Claude Code가 이미 존재하는 것들을 바탕으로 그것을 만들어 냅니다.

제자리에서 갱신되는 살아 있는 페이지

Claude Code가 아티팩트를 갱신하면 열려 있는 페이지가 제자리에서 새로고침되고, 팀원들은 갱신 내용이 게시되는 순간 그것을 확인할 수 있습니다. 게시할 때마다 같은 링크에 새 버전이 생기며, 버전 기록(version history)이 있어 언제든 이전 상태로 복원할 수 있습니다. 또한 갤러리(gallery)에서 지금까지 만든 모든 아티팩트를 둘러보고 관리할 수 있습니다.

내부 테스트에서 가장 흔했던 활용 사례 중 하나는 디버깅이었습니다. 일반적으로 다음과 같은 모습입니다. 한 엔지니어가 스탠드업(standup) 전에 장애 조사를 시작합니다. Claude Code는 로그를 훑어 내려가며 아티팩트를 게시합니다. 타임라인, 의심되는 커밋들, 그리고 오류율 차트가 담겨 있습니다. 그는 페이지 헤더에서 그 링크를 팀에 공유합니다. 스탠드업이 시작될 무렵, Claude는 조사가 진척됨에 따라 최신 정보를 반영하며 이미 두 번 다시 게시한 상태입니다. 아티팩트가 있으면 팀원과 이해관계자들이 "에이전트가 무엇을 찾았는지 설명해 달라"고 할 필요가 없습니다. 모두가 같은 컨텍스트로 같은 화면을 보고 있기 때문입니다.

조직에 한정된 비공개

모든 아티팩트는 기본적으로 작성자에게만 비공개입니다. 준비가 되면 페이지에서 바로 팀원과 조직에 공유할 수 있습니다. 아티팩트는 인증된 조직 구성원만 볼 수 있으며, 공개(public)로 만들 수는 없습니다. 관리자(admin)는 조직 수준 토글과 역할 기반 범위 지정(role-based scoping)으로 접근 권한을 관리하고, 보존 정책(retention policy)을 설정하며, 컴플라이언스 API(compliance API)를 통해 조직 전반의 가시성을 확보합니다.

시작하기

세션에 아티팩트를 요청하세요. 또는 그냥 시각적인 무언가를 요청해도 됩니다. 역할별 아이디어 몇 가지를 소개합니다.

  • 법무 / 오픈소스: 저장소에서 바로 가져온, 모든 의존성에 대한 라이선스 감사. 카피레프트(copyleft)를 표시해 줍니다. "모든 서드파티 의존성과 그 라이선스를 나열하고, 카피레프트인 것은 모두 표시하는 아티팩트를 만들어 줘."
  • 프라이버시: 개인정보가 코드 전반에서 어디에서 수집되고 저장되며 로깅되는지를 보여 주는 데이터 흐름도(data-flow map). "코드베이스 전반에서 개인정보를 다루는 지점을 추적해서 프라이버시 검토용 아티팩트로 만들어 줘."
  • 보안: 정확한 라인으로 연결되는 발견 사항들. 그래서 수정 지점이 명확합니다. "이번 검토에서 나온 인증(auth) 관련 발견 사항을 아티팩트로 만들고, 각 항목을 해당 코드에 연결해 줘."
  • FinOps / 플랫폼 재무: 인프라스트럭처-as-코드(infrastructure-as-code)에서 매핑한 클라우드 리소스와 비용 유발 요인. "Terraform에서 우리 클라우드 리소스를 서비스별로 묶고, 주요 비용 유발 요인과 함께 아티팩트로 매핑해 줘."
  • 소프트웨어 엔지니어: 리뷰어가 실제로 따라갈 수 있는 PR 또는 버그 설명. 디프(diff)와 그 주변 코드에서 끌어옵니다. "이 PR을 짚어 주는 아티팩트를 만들어 줘. 디프, 그 근거, 그리고 내가 무엇을 테스트했는지를 담아서."
  • 디자이너 & 프런트엔드 엔지니어: 한 화면에 대한 여러 UX 방향. 각각이 실제 컴포넌트로 만들어져 선택한 것을 바로 배포할 수 있습니다. "이 회원가입 폼의 UX 변형 5가지를 우리 컴포넌트 라이브러리로 만들어 아티팩트로 줘."
  • 스태프 엔지니어 & 아키텍트: 서비스가 실제로 어떻게 맞물려 있는지를 보여 주는 지도. 화이트보드가 아니라 실제 임포트 그래프(import graph)에서 그립니다. "결제(payments) 서비스가 어떻게 맞물려 있는지를 코드에서 가져와 아티팩트로 매핑해 줘."
  • SRE & 온콜(on-call): 조사를 진행할수록 커지고, 결국 사후 분석(postmortem)이 되는 장애 페이지. "이 장애를 아티팩트로 만들어 줘. 타임라인, 의심되는 커밋들, 우리 모니터링에서 나온 오류 급증을 담고, 내가 작업을 진행하는 대로 다시 게시해 줘."
  • 엔지니어링 매니저: 실제로 무엇이 출시됐는지를 보여 주는 페이지. 머지된 PR들로부터 만들어집니다. "이번 주 우리 팀에서 머지된 것들을 PR에서 가져와 프로젝트별로 묶어 아티팩트로 만들어 줘."

Claude Code가 페이지를 만들고 링크를 줍니다. 브라우저나 데스크톱 앱에서 열고, 헤더에서 공유하세요. 갱신 내용은 같은 URL에 자동으로 게시됩니다.

제공 범위

아티팩트는 Claude Team 및 Enterprise 조직을 대상으로 베타(beta)로 제공되며, Claude Code CLI와 데스크톱 앱에서 사용할 수 있고, 페이지는 어떤 브라우저에서도 볼 수 있습니다.
